# Checkpoint Verifier

A companion tool for the `checkpoint-builder` Rust crate that verifies checkpoint digests by computing them independently from blockchain data.

## Overview

The Checkpoint Verifier performs the following:

1. Connects to an Ethereum-compatible JSON-RPC endpoint (HTTP or WebSocket)
2. Fetches block data and transaction receipts for requested blocks
3. Computes merkle roots using the same algorithm as the attestation system
4. Chains checkpoint digests using `keccak256(blockNumber || merkleRoot || prevDigest)`
5. Either outputs computed checkpoints or verifies them against a CSV file

This tool is useful for:
- Validating that `checkpoint-builder` produces correct digests
- Debugging attestation/checkpoint mismatches
- Generating checkpoints from a different data source

## Prerequisites

- Node.js 18+
- Yarn package manager
- Access to an Ethereum-compatible JSON-RPC endpoint with `eth_getBlockByNumber` and `eth_getBlockReceipts` methods

## Installation

```bash
cd checkpoint-verifier
yarn install
yarn build
```

## Usage

### Verify Range

Compute and output the checkpoint digest for a range of blocks:

```bash
# Basic usage
yarn start verify-range \
  --rpc-url http://localhost:8545 \
  --start-block 0 \
  --end-block 100

# With starting digest (for continuing from a known checkpoint)
yarn start verify-range \
  --rpc-url http://localhost:8545 \
  --start-block 101 \
  --end-block 200 \
  --starting-digest 0x1234567890abcdef...

# With progress output
yarn start verify-range \
  --rpc-url http://localhost:8545 \
  --start-block 0 \
  --end-block 1000 \
  --verbose
```

**Options:**

| Option | Required | Description |
|--------|----------|-------------|
| `--rpc-url <url>` | Yes | HTTP (`http://`) or WebSocket (`ws://`, `wss://`) RPC URL |
| `--start-block <number>` | Yes | Starting block number (inclusive) |
| `--end-block <number>` | Yes | Ending block number (inclusive) |
| `--starting-digest <hex>` | No | 0x-prefixed 32-byte hex digest to chain from (default: zero hash) |
| `--verbose` | No | Show block-by-block progress |

**Output:**

```
Computing checkpoint for blocks 0 to 100...

Checkpoint at block 100:
  Digest: 0xabcdef1234567890...

CSV format: 100,0xabcdef1234567890...
```

### Verify CSV

Verify checkpoints from a CSV file generated by `checkpoint-builder`:

```bash
# Basic usage
yarn start verify-csv \
  --rpc-url http://localhost:8545 \
  --csv-path ../checkpoints_20260301T120000.csv

# With starting digest (if CSV doesn't start from genesis)
yarn start verify-csv \
  --rpc-url http://localhost:8545 \
  --csv-path ./checkpoints.csv \
  --starting-digest 0x1234567890abcdef...

# With verbose output
yarn start verify-csv \
  --rpc-url http://localhost:8545 \
  --csv-path ./checkpoints.csv \
  --verbose
```

**Options:**

| Option | Required | Description |
|--------|----------|-------------|
| `--rpc-url <url>` | Yes | HTTP or WebSocket RPC URL |
| `--csv-path <path>` | Yes | Path to the checkpoints CSV file |
| `--starting-digest <hex>` | No | Digest before the first checkpoint in CSV (default: zero hash) |
| `--verbose` | No | Show checkpoint-by-checkpoint progress and all digests |

**CSV Format:**

The expected CSV format is `block_number,digest` with no headers:

```csv
0,0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef
100,0xabcdef1234567890abcdef1234567890abcdef1234567890abcdef12345678
200,0x567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef12
```

**Output:**

```
Found 3 checkpoints in ./checkpoints.csv
Block range: 0 to 200

Verifying checkpoints...

============================================================
VERIFICATION RESULTS
============================================================

[PASS] Block 0
[PASS] Block 100
[FAIL] Block 200
  Expected: 0x567890abcdef...
  Computed: 0xfedcba098765...

============================================================
SUMMARY: 2/3 passed, 1 failed
============================================================
```

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | All checkpoints verified successfully |
| 1 | One or more checkpoints failed verification, or an error occurred |

## Algorithm

### Merkle Root Computation

For each block, the merkle root is computed from all transactions:

1. Each transaction is ABI-encoded along with its receipt
2. Leaf hash: `keccak256(0x00 || encoded_data)`
3. Inner hash: `keccak256(0x01 || left || right)`
4. Empty block: zero hash (32 bytes of zeros)

### Checkpoint Digest Chain

Checkpoint digests are chained using:

```
digest = keccak256(blockNumber_u64_be || merkleRoot_32bytes || prevDigest_32bytes)
```

Where:
- `blockNumber_u64_be`: Block number as big-endian 8 bytes
- `merkleRoot_32bytes`: Computed merkle root of the block
- `prevDigest_32bytes`: Previous checkpoint digest (or zero hash for genesis)
